1. Contexte du Projet

Ce projet s'inscrit dans le cadre d'un déploiement industriel massif de milliers de capteurs IoT[cite: 14]. L'objectif est de mettre en place une infrastructure Cloud Serverless hautement disponible sur AWS, capable de capter les flux de données envoyés par requêtes HTTP POST en temps réel, tout en fournissant un espace documentaire isolé et sécurisé aux équipes de Data Science[cite: 15, 16, 17].

2. Description de l'Architecture Big Data

L'intégralité de l'infrastructure est entièrement automatisée via AWS CloudFormation (SAM) et s'articule autour de deux sous-systèmes principaux[cite: 17]:

A. Le Pipeline d'Ingestion Analytique (Serverless)

  • Amazon CloudFront : Agit comme premier CDN global pour collecter les flux IoT au plus près des capteurs avec une latence minimale[cite: 17, 23].
  • Amazon API Gateway : Point d'entrée HTTPS qui reçoit les requêtes POST et les transfère à la couche de calcul[cite: 15, 23].
  • AWS Lambda (Python 3.11) : Traite les données à la volée, calcule la température moyenne et compte les anomalies de statut[cite: 17, 39, 43].
  • Amazon S3 (Data Lake Brut) : Stocke l'historique complet et immuable des payloads bruts sous forme de fichiers JSON, organisés selon une stratégie de partitionnement temporel strict (year=YYYY/month=MM/)[cite: 17, 40, 41].
  • Amazon DynamoDB (Serving Layer) : Fait office de Feature Store analytique en stockant les rapports condensés d'exécution indexés par l'ID de requête pour des lectures ultra-rapides[cite: 17, 44, 45].

B. Le Portail Documentaire Dédié

La documentation technique est stockée de manière isolée dans un bucket S3 privé[cite: 19]. L'accès public est strictement bloqué à la racine, et la consultation se fait exclusivement à travers une seconde distribution Amazon CloudFront configurée avec le protocole Origin Access Control (OAC), garantissant une sécurité optimale[cite: 19, 27].
